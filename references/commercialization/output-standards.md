# Commercialization Output Standards

## Tone

Use a direct management-consulting style. Avoid generic innovation language.

## Structure

Prefer tables when comparing projects, directions, risks, roles, or scenarios.

Every major output should include:

- what is known
- what is inferred
- what is missing
- what decision the output supports

## Handoff

Each specialist stage ends with:

```markdown
## Handoff

Next owner:
Next input to read:
Key conclusions:
Caveats:
Missing data:
Risks:
Required next action:
```

## Language

Operational skill and workflow instructions are in English.

User-facing deliverables should follow the user's language. For Kirill's Russian strategy work, default to Russian unless the user asks otherwise.

## Dossier Updates

When updating `working-dossier.md`, preserve existing sections. Add or revise only the stage-owned content and status fields needed for the workflow.
