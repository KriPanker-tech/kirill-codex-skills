# Commercialization Evidence Rules

## Core Rule

Do not invent data. If a field is missing, write `data insufficient`.

## Evidence Types

Use these labels consistently:

- `document fact`: directly supported by provided documents
- `external benchmark`: supported by cited external material
- `expert hypothesis`: reasoned interpretation that needs validation
- `missing data`: required information not available in the current source set

## Numeric Claims

For EBITDA, EMV, market size, margins, capex, timing, or probability:

- use only sourced numbers
- preserve units and time horizons
- mark assumptions explicitly
- do not convert a qualitative opportunity into a quantified claim unless the math is provided

## Market Data

Use external market data only when the task asks for it or the workflow stage requires it. External data should strengthen reasoning, not replace project evidence.

## User Questions

Ask the user when missing data changes the decision:

- target EBITDA / EMV
- project ownership
- investment constraints
- production availability
- customer validation
- regulatory or IP blocker
- board / management audience requirements
