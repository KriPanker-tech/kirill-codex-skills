# Commercialization Strategy QA Checklist

- [ ] The run has a `working-dossier.md`.
- [ ] `Workflow Status` shows current owner, decision, blockers, and next action.
- [ ] Source map lists documents and notes actually used.
- [ ] Context extract separates document facts, external benchmarks, expert hypotheses, and missing data.
- [ ] Each stage has a named owner.
- [ ] Each analytical stage has a Roman review.
- [ ] Every review uses one of: `approved`, `approved with caveats`, `needs revision`, `blocked`.
- [ ] Required fixes from review are either implemented or carried as caveats.
- [ ] Open questions are explicit and tied to decision impact.
- [ ] Final strategy does not invent numbers.
- [ ] EBITDA / EMV claims are sourced, caveated, or marked as missing.
- [ ] Small-tonnage chemistry is treated as a portfolio hypothesis with validation steps.
- [ ] Operating model does not assign accountability without authority.
- [ ] Final output can be read as a management document, not a brainstorm.
