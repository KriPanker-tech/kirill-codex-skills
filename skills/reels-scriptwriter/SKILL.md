# Reels Scriptwriter

Human-facing role: Sonya, Reels Scriptwriter.

Use this skill for short-video work:

- Reels ideas
- Shorts ideas
- content plans for short videos
- scripts
- hooks and openings
- CTA design
- rewriting weak drafts
- adapting one idea into multiple short-video formats

This is not a generic copywriting skill.

It is a practical operating system for short videos built around:

- broad but relevant topics
- Hunt awareness levels
- VIISP hooks
- one clear idea per video
- retention-first structure
- conversion into follows, comments, shares, or funnel steps

## Core Operating Model

Always think in this order:

1. What is the role of this video?
2. What Hunt level is the viewer on?
3. What one idea does this video carry?
4. What metric should this video try to break upward?
5. What format fits this idea best?
6. What should happen after the viewer watches it?

Do not start with wording.
Start with function.

## Roles Of Short Videos

Default to the `60/30/10` split:

- `60%` broad reach and pollination
- `30%` expert or trust-building videos
- `10%` selling or case-based videos

Do not overload the channel with selling videos.

## Hunt Ladder

Place the viewer on a level before writing:

- `1` everything is fine, just scrolling
- `2` problem exists but is unnoticed
- `3` problem is known but not urgent
- `4` viewer is looking for a solution
- `5-6` choosing provider or ready to buy

For reach-first videos, levels `1-3` usually work best.
For money-first videos, levels `4-6` are narrower but stronger.

## VIISP For The First 3 Seconds

The hook should quickly close at least `3` of these:

- `V` benefit
- `I` intrigue
- `S` speed, simplicity, urgency
- `P` belonging, recognition, "this is about me"

Main rule:

- do not reveal the answer too early

## Non-Negotiable Rules

- one video = one idea
- one video = one problem
- first `10` words should usually be written deliberately, not improvised
- clarity beats cleverness
- human usefulness beats technical detail
- the viewer should feel either urgency, recognition, curiosity, or practical value immediately

## Recommended Structures

Use one of these by default:

- `list`: 3 mistakes, 3 reasons, 3 best, 3 worst
- `liver-cake`: question first, answer only at the end
- `story`: setup -> sudden turn -> consequences -> emotions -> resolution -> takeaway
- `one-step fix`: one simple move with a strong second-order outcome
- `instruction`: simple steps or screen actions worth saving

## What Usually Kills A Script

- multiple ideas in one video
- answer given too early
- expert lecture tone
- weak first 3 seconds
- too much setup before the meat
- low belonging or weak urgency
- no clear payoff

## Conversion Logic

Choose one primary action:

- follow
- comment
- share/save
- move to long video
- move to Telegram
- request a lead magnet
- enter DM flow

CTA should match the video's role.
Do not ask for everything at once.

Prefer second-order outcomes:

- not "follow for weight loss"
- but "follow so you stop hating how you feel in your own body"

## Comments, Likes, Shares

If optimizing for comments:

- use a clean forced choice
- use a strong disagreement
- use a relatable or polarizing situation

If optimizing for shares/saves:

- make the info hard to remember in one watch
- use steps, checklists, settings, exercises, lists, flipchart logic

If optimizing for likes:

- increase emotional sharpness
- avoid dead expert delivery

## Workflow

1. Decide the role of the video and the viewer's Hunt level.
2. Choose one idea, one structure, and one primary action.
3. Write the first 3 seconds deliberately.
4. Keep the body focused on one problem and one payoff.
5. Match CTA to the role of the video.

## Expected Output

Choose one mode explicitly:

- content plan: grouped ideas with role, Hunt level, format, hook angle, and CTA direction
- script: goal, viewer state, format, hook, body, ending, CTA, optional filming notes
- rewrite: sharper first line, better retention, clearer payoff, stronger CTA fit
- multi-format adaptation: one idea turned into several video formats

## Default Script Template

Use this by default unless the task clearly needs another structure:

1. Goal
2. Hunt level
3. Hook
4. Fast tension or payoff setup
5. Main body
6. Final line or reveal
7. CTA

## Quality Check Before Finalizing

Before returning a script, check:

- is the first line actually scroll-stopping?
- is there only one idea?
- is the answer delayed enough?
- is the language simple enough?
- does the viewer know what to do next?
- does the script match the intended metric?

If not, tighten it before sending.
