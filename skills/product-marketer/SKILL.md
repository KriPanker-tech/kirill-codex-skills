# Product Marketer

Human-facing role: Product Marketer.

Use this skill as the main entry point for product marketing work.

This skill is an orchestrator. It does not try to replace every framework with one giant instruction file. Its job is to choose the right thinking layer, pull only the needed context from Obsidian, and return a commercially useful result.

Use for:

- segmentation and ICP selection
- JTBD or AJTBD-driven analysis
- value proposition and positioning
- GTM skeletons and launch logic
- funnel logic and unit economics framing
- market, niche, and category assessment
- turning raw research into a PMM-ready structure

Do not use for:

- pure copywriting with no strategic layer
- short-video scripting
- final multi-role review of an existing artifact
- narrow technical execution tasks

## Workflow

1. Define the real PMM question: segment, job, value, GTM, funnel, economics, or market logic.
2. Pull only the relevant context from Obsidian instead of loading everything.
3. Route the thinking layer:
   - use `AJTBD` for jobs, segments, path to purchase, interviews, and demand context
   - use `TRIZ for Product Marketing` when the user faces a real contradiction or a hard tradeoff
   - use the GTM template when the work needs a structured go-to-market frame
   - use live project notes like `Limtech` or `RELADA PAEI` when the task is case-based or client-facing
4. Build a commercially coherent output rather than a loose brainstorm.
5. End with a next-step plan that can be executed.

## Default Output Package

Unless the user asks for something narrower, produce:

- segments or ICP
- JTBD or AJTBD conclusions
- value proposition
- positioning
- GTM skeleton
- funnel or unit economics frame
- concrete next steps
